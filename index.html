Privacy Policy – Face-T Extension

Last updated: April 23, 2026

1. Overview

Face-T Extension is an enterprise-use browser extension designed to help organizations analyze UX flows within their own web applications. On approved enterprise web pages, the extension records certain user interaction events and captures before-and-after page screenshots to support UX analysis.

This policy explains what data the extension handles, how it is used, where it is sent, and how it is retained.

2. Scope and Approved Pages

Face-T Extension is intended for internal enterprise use.

The extension checks whether the current page is permitted by an organization-managed allow-list. Capture and processing occur only on pages approved by that policy as part of the Face-T workflow.

3. Data Collected

  • Web page URLs – used to verify whether the current page is on the organization’s allow-list policy.
  • User interaction events – including click events and Enter key press events, used to identify the timing of screenshot capture.
  • Page screenshots (t0 / t1) – a “before” screenshot is captured at the moment of interaction, and an “after” screenshot is captured once the page finishes rendering.
  • Extension settings – including backend URL and local agent URL values required for operation in the enterprise environment.

4. How Data Is Used

All handled data is used solely for the Face-T workflow, including:

  • verifying whether a page is approved for capture;
  • correlating user interactions with before-and-after screenshots;
  • analyzing UX navigation patterns and page transitions within the organization’s web application;
  • supporting diagnostics and operational troubleshooting.

The data is never used for advertising, profiling, or any purpose unrelated to the extension’s enterprise UX analysis function.

5. Data Transmission and Local Processing

All captured events and screenshots are transmitted exclusively to a locally running Face-T agent on the user’s own machine. The default local endpoint is:

http://127.0.0.1:17777

The extension itself does not send captured screenshots or interaction event data directly to external servers, cloud analytics services, or unrelated third parties.

6. Security

Face-T Extension is designed to limit data handling to the approved enterprise workflow described in this policy.

  • screenshot and interaction data are delivered to a local Face-T agent running on the same device;
  • the extension stores only the settings needed for operation, such as backend URL and local agent URL values;
  • the extension does not persist screenshots or interaction event data as part of its own long-term storage.

7. Data Retention

The extension itself does not store screenshots or interaction event data persistently beyond extension settings needed for operation.

Retention of screenshot and event data after delivery is managed by the local Face-T agent and backend under the control of the enterprise operator.

8. Third-Party Sharing

We do not sell, trade, or transfer user data to advertisers, data brokers, or unrelated third parties.

Data handled by the extension is used only for the enterprise UX analysis workflow described in this policy.

9. Enterprise Administration

Face-T Extension operates according to organization-managed configuration, including approved page scope, backend settings, and local agent settings. Users should contact their organization or administrator for questions about deployment, approved pages, or operational settings.
