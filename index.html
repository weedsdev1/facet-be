Privacy Policy – Face-T Extension

Last updated: April 23, 2026

1. Overview

Face-T Extension is an enterprise-use browser extension designed to help organizations analyze UX flows within their own web applications. On enterprise-approved web pages, the extension records certain user interaction events and captures before-and-after page screenshots to support UX analysis.

This policy explains what data the extension handles, how it is used, where it is sent, and how it is protected.

2. Scope and Approved Pages

Face-T Extension is intended for internal enterprise use.

The extension checks whether the current page is permitted by an enterprise-managed allow-list. Capture and processing occur only on pages approved by that policy. The extension is not intended to capture data from arbitrary websites outside the approved enterprise workflow.

3. Data Collected

  • Web page URLs – used to determine whether the current page is permitted by the organization’s allow-list policy.
  • User interaction events – including click events and Enter key press events, used to identify the timing of UX captures.
  • Page screenshots – including a “before” screenshot captured at the time of interaction and an “after” screenshot captured after the page finishes updating.
  • Extension configuration values – such as backend URL and local agent endpoint settings, required for the extension to operate in the enterprise environment.

4. How Data Is Used

The data handled by Face-T Extension is used solely for the following purposes:

  • determining whether a page is approved for the Face-T workflow;
  • correlating user interactions with before-and-after screenshots;
  • analyzing UX navigation and page transition behavior within the organization’s web applications;
  • supporting troubleshooting and operational diagnostics for the Face-T workflow.

The data is not used for advertising, marketing, user profiling, or any purpose unrelated to the extension’s enterprise UX analysis function.

5. Data Transmission and Local Processing

Captured interaction events and screenshots are forwarded only to a Face-T local agent running on the same device as the browser. The default local endpoint is:

http://127.0.0.1:17777

The extension is designed to send captured data to a local loopback endpoint on the user’s device as part of the Face-T workflow. The extension itself does not transmit captured screenshots or interaction event data directly to external third-party services, advertisers, or public cloud analytics services.

6. Security

Face-T Extension is designed to limit data handling to the approved enterprise workflow described above.

  • captured screenshots and interaction events are sent to a locally running Face-T agent on the same device;
  • the extension stores only configuration values needed for operation, such as backend and agent endpoint settings;
  • the extension does not persist screenshots as part of its own long-term storage.

Security controls for any processing after local delivery are managed by the organization operating the Face-T environment.

7. Data Retention

The extension itself does not permanently retain screenshots or interaction event data. It stores only the configuration values necessary for operation.

Retention of screenshots, event data, or any related analysis records after delivery to the local Face-T agent is managed by the organization’s Face-T environment and enterprise operator.

8. Third-Party Sharing

We do not sell, trade, or transfer collected data to advertisers, data brokers, or unrelated third parties.

Data handled by the extension is used only for the enterprise UX analysis workflow described in this policy.

9. Enterprise Administration

Face-T Extension operates according to organization-managed configuration, including approved page scope, backend settings, and local agent settings. Users should contact their organization or administrator for questions about deployment, approved pages, or operational settings.
