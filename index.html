Privacy Policy – Face-T Extension

Last updated: April 23, 2026

1. Overview

Face-T Extension is an enterprise-use browser extension designed to help organizations analyze UX flows within their own web applications. On approved enterprise web pages, the extension records certain user interaction events and captures before-and-after page screenshots to support UX analysis.

This Privacy Policy explains what data the extension handles, how that data is used, where it is sent, how long it is retained, and how the extension limits its own handling of captured data.

2. Scope and Approved Pages

Face-T Extension is intended for internal enterprise use.

The extension checks whether the current page is permitted by an organization-managed allow-list. Capture and processing occur only on pages approved by that policy as part of the Face-T workflow. The extension is not intended to capture data from arbitrary websites outside the approved enterprise workflow.

3. Data Collected

  • Web page URLs – used to determine whether the current page is permitted by the organization’s allow-list policy.
  • User interaction events – including click events and Enter key press events, used to identify the timing of UX capture.
  • Page screenshots (t0 / t1) – including a “before” screenshot captured at the moment of interaction and an “after” screenshot captured after the page finishes updating.
  • Extension configuration values – including backend URL and local agent endpoint settings required for operation in the enterprise environment.

4. How Data Is Used

Data handled by Face-T Extension is used solely for the Face-T workflow, including:

  • verifying whether a page is approved for capture;
  • correlating user interactions with before-and-after screenshots;
  • analyzing UX navigation patterns and page transitions within the organization’s web applications;
  • supporting diagnostics and operational troubleshooting.

Data handled by the extension is not used for advertising, marketing, profiling, or any purpose unrelated to the extension’s enterprise UX analysis function.

5. Data Transmission and Local Processing

The extension forwards captured interaction events and screenshots only to a Face-T local agent running on the same device as the browser. The default local endpoint is:

http://127.0.0.1:17777

The extension is designed to send captured data only to a local loopback interface on the user’s device. The local agent endpoint is intended to be accessible only from that device and is not intended to be exposed to remote network access as part of the extension’s normal operation.

The extension itself does not directly transmit captured screenshots or interaction event data to advertisers, data brokers, public cloud analytics services, or unrelated third-party services.

Any processing that occurs after delivery to the local Face-T agent is governed by the enterprise environment operated by the organization.

6. Security

Face-T Extension is designed to limit its own data handling to the approved enterprise workflow described in this policy.

  • captured screenshots and interaction events are delivered to a local Face-T agent running on the same device;
  • the extension stores only the settings needed for operation, such as backend URL and local agent endpoint values;
  • the extension does not persist screenshots or interaction event data as part of its own long-term storage;
  • the extension does not use captured data for advertising, sale, or unrelated third-party disclosure.

7. Data Retention

The extension itself does not permanently retain screenshots or interaction event data beyond temporary handling required for the Face-T workflow. The extension stores only configuration values needed for operation.

Retention of screenshot and event data after delivery is managed by the local Face-T agent and any connected enterprise systems under the control of the organization operating the Face-T environment.

8. Third-Party Sharing

We do not sell, trade, or transfer data handled by the extension to advertisers, data brokers, or unrelated third parties.

Data handled by the extension is used only for the enterprise UX analysis workflow described in this policy.

9. Enterprise Administration

Face-T Extension operates according to organization-managed configuration, including approved page scope, backend settings, and local agent settings. Users should contact their organization or administrator for questions about deployment, approved pages, or operational settings.
